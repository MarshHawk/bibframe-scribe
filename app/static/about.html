About the BIBFRAME Editor prototype

Overview

The BIBFRAME editor prototype is a tool that demonstrates how to catalog various materials in native BIBFRAME. As a linked data framework on the web, BIBFRAME is designed to integrate with and engage in the wider information community while also serving the very specific needs of local implementation.

This prototype is designed to demonstrate how one would catalog entities in BIBFRAME, how the workflow is integrated with web services, and how to move between multiple types of entities in a single tool. BIBFRAME is designed to work with existing standards (RDA, DACS, VRA, etc.) and yet-to-be-developed community vocabularies. For demonstration purposes, this prototype uses initial drafts for a small set of material types.

Status: Pre-release demonstration

Objectives

  • Demonstrate how components of the BIBFRAME framework come together in an editing platform.
  • Provide a method to examine workflows for cataloging in native BIBFRAME.
  • Explore common relationships between Works such as translations, derivative works, etc.
  • Demonstrate how different vocabularies are accommodated in a single interface.
  • Explore the ability to link to external services that are needed in workflows, such as VIAF, Library of Congress.
  • Demonstrate unique data entry features such as “look ahead” searches of remote services and a drag and drop image zone for images.
  • Show possible future functionality such as user activity dashboard via mock-up pages.

Prototype Functionality Facts

  • Seeded with 15 records converted from MARCXML.
  • Interface reflects a subset of the BIBFRAME vocabulary. The interface guides you to the particular instance and brings up only fields that are pertinent to that instance type.
  • Local search for instances and all other resources
  • Uses remote search services:
    • Library of Congress Linked Data Service (names, subjects, languages, places
    • assignFAST (subjects)
    • VIAF (names, subjects)
    • AGROVOC (subjects)
